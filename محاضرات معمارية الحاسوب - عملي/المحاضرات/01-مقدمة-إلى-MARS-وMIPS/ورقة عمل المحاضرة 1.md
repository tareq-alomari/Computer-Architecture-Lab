## القسم الصفري: تجهيز البيئة — خطوات مفصلة من دليل شرح العمل على MARS

اتبع الخطوات التالية بالترتيب (موافقة لدليل د. خالد القحصة):

### المهمة 1: فك ضغط حزمة الأدوات CA-Tools.rar
1. افتح مجلد `أدوات/` في المستودع
2. ستجد ملف `CA-Tools.rar`
3. استخدم WinRAR أو 7-Zip لفك الضغط
4. تأكد من ظهور المجلد `CA-Tools/` الذي يحتوي على:
   - `jre-7u7-windows-i586.exe` — مشغل Java
   - `Mars4_3.jar` — برنامج MARS
   - `logisim-win-2.7.1.exe` — محاكي Logisim
   - `ECA-Tools/` — أدوات إضافية

### المهمة 2: تثبيت Java Runtime
1. افحص إذا كان Java مثبتاً مسبقاً:
   ```bash
   java -version
   ```
2. **إذا ظهر رقم الإصدار ← Java موجود، انتقل إلى المهمة 3**
3. إذا لم يكن Java مثبتاً:
   - **Windows:** اذهب إلى مجلد CA-Tools ← شغّل `jre-7u7-windows-i586.exe`
   - **Linux (Ubuntu/Debian):** `sudo apt update && sudo apt install default-jre`
   - **macOS:** `brew install --cask java`
4. بعد التثبيت، تحقق مرة أخرى:
   ```bash
   java -version
   ```
   يجب أن يظهر إصدار Java (مثلاً `1.7.0_07` أو أحدث)

### المهمة 3: تشغيل MARS
1. اذهب إلى مجلد `CA-Tools/` الذي فككت ضغطه
2. شغّل MARS بإحدى الطريقتين:
   ```bash
   # الطريقة 1 — سطر الأوامر:
   java -jar Mars4_3.jar

   # الطريقة 2 — نقر مزدوج على Mars4_3.jar
   ```
3. ستظهر نافذة MARS الرئيسية

### المهمة 4: التعرف على واجهة MARS

| النافذة | الوظيفة |
|---------|---------|
| **Edit** | محرر النصوص — اكتب كود MIPS هنا |
| **Registers** | تعرض المسجلات (الاسم، الرقم، القيمة) |
| **Text Segment** | الكود المُجمّع + Address + Code + Basic + Source |
| **Data Segment** | البيانات في الذاكرة (Address, Value بصيغة Hex/ASCII) |
| **Run I/O** | نافذة الإدخال والإخراج |
| **MARS Messages** | رسائل الأخطاء |
| **Coproc 0** | مسجلات المعالج المساعد (الاستثناءات) |
| **Coproc 1** | مسجلات الفاصلة العائمة (FPU) |

### المهمة 5: فتح مثال وتشغيله (كما في الدليل)
1. من قائمة **File → Open**
2. اختر الملف `mips001.asm` الموجود في مجلد المحاضرة
3. **F3 → Assemble** — لتجميع الكود
4. **F5 → Execute (Run)** — لتشغيل البرنامج
5. أدخل أعداداً في نافذة **Run I/O** ولاحظ النتائج
6. جرب **F7 → Step** — للتنفيذ سطراً سطراً

### المهمة 6: اختبر البيئة ببرنامج بسيط
1. من قائمة **File → New**
2. اكتب البرنامج التالي:
```mips
.data
msg: .asciiz "MARS is working!"
.text
main:
  li $v0, 4
  la $a0, msg
  syscall
  li $v0, 10
  syscall
```
3. اضغط F3 → **Assemble**
4. اضغط F5 → **Run**
5. تأكد من ظهور "MARS is working!" في نافذة **Run I/O**

✅ **إذا ظهرت الرسالة، فبيئتك جاهزة!**

### أدوات التصحيح (Debugging) — من الدليل:
- **Step (F7):** تنفيذ تعليمة واحدة كل ضغطة — لمشاهدة تغير المسجلات خطوة بخطوة
- **Breakpoint (Bkpt):** انقر على عمود Bkpt في Text Segment لتعيين نقطة توقف
- **Go:** تشغيل البرنامج حتى نقطة التوقف
- **Reset:** إعادة تعيين المحاكي لحالة البداية

## القسم الثاني: تمارين برمجية

### التمرين 1: أول برنامج في MARS
اكتب برنامج MIPS يقوم بما يلي:
1. طباعة الرسالة `"Hello, MIPS!"`
2. قراءة عدد صحيح من المستخدم
3. طباعة العدد المُدخل
4. إنهاء البرنامج

**إرشادات:** استخدم `li` لتحميل قيم syscall في `$v0`.

### التمرين 2: التعامل مع المسجلات
اكتب برنامجًا يقوم بـ:
1. تحميل القيمة `15` في المسجل `$t0`
2. تحميل القيمة `25` في المسجل `$t1`
3. نسخ قيمة `$t0` إلى `$t2`
4. جمع `$t1` و `$t2` ووضع الناتج في `$t3`
5. طباعة قيمة `$t3`

### التمرين 3: محاكاة العمليات
اكتب برنامجًا يقرأ رقمين صحيحين من المستخدم، ثم يقوم بطباعة مجموعهما والفرق بينهما.

---

### تمرين إضافي: برنامج الجمع والتكرار (مستوحى من مثال mips001.asm)
افتح الملف `mips001.asm` في MARS وحلله، ثم أجب:
1. ما وظيفة المسجل `$t2` في البرنامج؟
2. ماذا تفعل التعليمة `beq $t2, $zero, done_input`؟
3. كم مرة سينفذ التكرار (loop) إذا أدخل المستخدم الأعداد 5, 10, 3, 0؟
4. **عدّل البرنامج** ليحسب أيضاً متوسط الأعداد المُدخلة (المجموع ÷ العدد).

## القسم الثالث: سؤال التحدي

### التحدي 🏆
بدون استخدام تعليمة `move` أو `add` مع `$zero`، اكتب برنامجًا ينفذ تبادل (swap) لقيم مسجلين `$t0` و `$t1` باستخدام 3 تعليمات فقط (XOR technique). ثم اشرح كيف يعمل الكود خطوة بخطوة.

**مثال:**
```
قبل التنفيذ: $t0 = 5, $t1 = 7
بعد التنفيذ: $t0 = 7, $t1 = 5
```

---

**💡 تلميح:** ابحث عن تعليمة `xor` واستخدمها مع المسجلات.
